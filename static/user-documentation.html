Smart Assistant

This is a smart assistnat that is powered by a LLM (similar model to what is present in ChatGPT) to help answer the questions you have. The AI has access to the database of documentat that it will use to help answer your questions. You can ask it anything related to the investigation reports from TAIC, ATSB and TSB. It does also have powerful reasoning capabilities so can be used to help with jobs that you might ask another AI chatbot. I would recommend talking to it like a colleague rather than a search engine to get the best results. Try to be specific with your questions to get the best answers.

How to use

To use the smart assistant simply type your question into the input box and hit the "Send" button. The AI will then process your question and provide an answer. You can have a back-and-forth conversation with the AI by asking follow-up questions. Each conversation is stored in the chat history on the right side of the screen. Note: Each conversation is unique and the AI will not remember previous conversations.

Editing, Undoing

If you asked it a question and are not happy with the answer you can either:

  • Click the "Undo" button (Small circle button on the bottom left of last AI response) to edit the previous sent question and refine it to be more specific.
  • Click the "Edit" button (Small pencil icon on the bottom right of your messages) to modify a previous message. You need to press the tick after editing to save the changes. This will forget the part of the conversation after that message and regenerate a new response.
  • Send a follow up message to clarify what you wanted it to do.

Managing conversations

You can manage your conversations using the buttons above the chat box, or in the side bar on the left:
Note that all conversations are autosaved by default.

  • New Conversation - Starts a new conversation and clears the chat history.
  • Delete Conversation - Deletes the current conversation from history.
  • Load Conversation - Loads a previously saved conversation from history.

Underlying database

Key to the smart tools platform is a database that is generated by scarping the public websites of TAIC, ATSB and TSB and processing the PDF reports. This database is then used to power both the smart search and smart assistant features.

Document types

In the database there are 4 types of the documents:
  • section - Each report is divided into numbered sections (if possible), otherwise simply split it paragraph sized chunks of text
  • summary - For ATSB and TAIC there are investigation summaries present on the public webpage. These are normally similar too but not always identical to the executive summaries.
  • safety issue - For ATSB the safety issues are scrapped directly from website. For TSB and TAIC they are extracted from the report so are not guaranteed to be identical, in cases where reports don't have safety issues and the AI deems a safety issues was present according to current defintion it will infer the safety issue
  • recommendation - All recommendations are either extracted verbatium from the reports or scraped from webpages.

Year

The year of the documents are related to the occurrence date of the accidents.

IDs

Each document in the database has a unique ID that is associated with it. It is called the "document_id". A document will be one of the four types listed above (section, summary, safety issue, recommendation). Furthermore each document is associated with a single investigation report which also has its own unique ID called the "report_id", which also is linked to the "agency_id" that is the ID the agency uses to identify the report.

Smart Search

This is a semantic search engine of the database. You conduct searches in the same way as a normal search engine. Type your query in the search box and press enter. You can use the filters to narrow down the results.
Note: seraching if you see nothing happen make sure to scroll to the bottom of the page to see the progress bar as it can take a few seconds to search.

Relevance filters

When doing a semantic serach the results are ranked by relevance to the query. With 1 being the most relevant and 0 being the least relevant. It is default set to 0.2 to filter out all of the irrelevant results. You can adjust this to your liking if you are not getting enough results or want to see more results. It can be really sensitive to small changes so try adjusting it in increments of 0.05.

Graphs

The search results page also includes graphs to help visualise the results. You can see the distribution of document types and years in the results. This can help you identify trends and patterns in the data. Click on the 'Results graph' drop down just above the results to view these visualizations. Each of them can be downloaded as an image by clicking the camera icon in the top right of the graph.

Post search searching

After performing a search, you can further refine your results using the post-search searching. In the search bar at the top right of the results table you can type in words and it will do an exact word match filter on the results already returned from the semantic search. This is useful for narrowing down results to specific topics or keywords after the initial semantic search has been performed.

Regular text search

You can also do a regular text search by enclosing your search terms in double quotes. For example: "fuel starvation". This will return all results that contain the text "fuel starvation" regardless of the semantic meaning.
